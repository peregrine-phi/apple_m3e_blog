## 设计系统对比度风险评估
- 产出 `DESIGN_TOKEN_CONTRAST_AUDIT.md` — 从 6 个维度评估当前 ~238 个 token 的对比度安全性
- **关键发现**：Primary button light 对白字 ≈ 3.7:1（低于 WCAG AA 4.5:1）、Secondary ≈ 3.4:1 — 为最严重对比度违规
- 文本层级使用透明度（0.55/0.28/0.12）导致对比度不可静态验证
- 缺少 success/warning 的 on-* 配对和 dark 模式覆写
- 组件级 token 命名不一致（--btn-* vs --chip-* vs --md-btn-*），建议统一为 --blog-comp-*
- filters.css 15 处 !important 破坏 token 调控能力
- 完全缺失自动化对比度校验（构建时/CI/运行时均无）
- 提出 5 层防线架构 + P0-P2 优先级行动清单
